# 🎓 JARVIS 新機能ガイド - 高校生でもわかる解説

## 🤔 そもそもJARVISって何？

JARVISは「研究論文を自動で読んで、まとめて、評価してくれるAIシステム」です。

例えるなら...
- **図書館の司書さん** + **研究者の助手** + **レポートを書いてくれる秘書**
- 全部がAIになったようなもの！

---

## 🧠 EPIC-0.5: Lyra（リラ）- 頭脳担当

### 何をする？
**「AIを管理するAI」** です。

### わかりやすく言うと...
学校の先生が生徒の宿題をチェックするように、Lyraは他のAIの仕事をチェックします。

```
「この説明、根拠がないよ？」
「ここ曖昧だから書き直して」
「これで完璧！OK」
```

### 4Dメソッド
1. **Deconstruct（分解）** - 問題を細かく分ける
2. **Diagnose（診断）** - 何が足りないか見つける
3. **Develop（開発）** - 改善した指示を作る
4. **Deliver（配布）** - 他のAIに仕事を渡す

## 📋 EPIC-0: Contracts（契約）- ルールブック

### 何をする？
**「全員が同じフォーマットで話す」** ためのルール。

### わかりやすく言うと...
みんなが同じ用紙を使ってレポートを書くようなもの。

| 用語 | 意味 |
|------|------|
| TaskContext | 「何をしたいか」を書く欄 |
| Artifacts | 「集めたデータ」を入れる箱 |
| ResultBundle | 「結果と証拠」をまとめたファイル |
| EvidenceLink | 「これが証拠です」という印 |

### 重要ルール
> **「根拠のない主張は禁止！」**

テストで「答えは○○です」と書くだけじゃダメで、「なぜなら〜だから」と理由を書くのと同じ！

## 🔍 EPIC-2: Retrieval（検索）- 資料集め担当

### 何をする？
論文を **検索して見つける**。

### 機能たち

| 機能 | 何をする？ | 例 |
|------|----------|-----|
| QueryExpander | 検索ワードを増やす | 「がん」→「がん、腫瘍、悪性新生物」 |
| BM25Retriever | キーワードで検索 | 図書館の検索システム |
| SectionEmbedder | 内容を数値化 | 文章を「0.1, 0.5, 0.3...」という数字に |
| DuplicateDetector | 重複を消す | 同じ論文が2回出たら1つに |

### イメージ
「免疫療法」と検索すると...
1. 関連ワードも自動で追加
2. 何千もの論文から候補を絞る
3. 重複を削除
4. 最も関連性が高い20件を選ぶ

---

## 📝 EPIC-3: Extraction（抽出）- 情報取り出し担当

### 何をする？
論文から **大事な情報だけ抜き出す**。

### 抽出するもの

| 種類 | 何を取り出す？ | 例 |
|------|--------------|-----|
| Claim | 論文の主張 | 「この薬は効果がある」 |
| Evidence | 証拠の場所 | 「3ページ目の表1が根拠」 |
| Numeric | 数字データ | 「n=100人、p<0.05」 |
| Methods | 実験方法 | 「Western blotを使用」 |
| Limitation | 弱点 | 「サンプル数が少ない」 |

### イメージ
論文という「分厚い本」から、
- 重要な結論だけ
- 数字のデータだけ
- 注意点だけ

をハイライトしてくれる！

## 📊 EPIC-4: Summarizer（要約）- まとめ担当

### 何をする？
長い論文を **短くまとめる**。

### 3つのモード

| モード | 長さ | 用途 |
|-------|------|------|
| ショート | 300字 | Twitter感覚で読みたい時 |
| ミディアム | 1000字 | ちゃんと理解したい時 |
| 章別 | 各章ごと | 詳しく知りたい時 |

### 特別機能
- **比較要約**: 複数の論文を比べる（「AとBの違いは〜」）
- **初学者モード**: 専門用語をわかりやすく言い換え

| 専門用語 | わかりやすく |
|---------|------------|
| apoptosis | 細胞の自然死（プログラムされた死） |
| in vitro | 試験管内の実験 |
| placebo | 偽薬（効果のない比較用の薬） |

## ⭐ EPIC-5: Scoring（採点）- 評価担当

### 何をする？
論文に **点数をつける**。

### 5つの評価軸

| 評価 | 何を見る？ | イメージ |
|------|----------|---------|
| 重要度 | どれくらい重要か | 「超重要」〜「普通」 |
| 信頼度 | 証拠があるか | 「確実」〜「怪しい」 |
| バイアス | 偏りがないか | 「公平」〜「偏ってる」 |
| エビデンス階層 | 研究の質 | 「臨床試験」>「動物実験」 |
| 学習ROI | 読む価値 | 「絶対読むべき」〜「後回し」 |

### エビデンス階層（研究の信頼度ランキング）
```
🥇 メタ分析・システマティックレビュー（最強）
🥈 ランダム化比較試験（RCT）
🥉 コホート研究
4位 症例対照研究
5位 症例報告
6位 専門家の意見
```

## 🕸️ EPIC-6: Knowledge Graph（知識グラフ）- 関係図担当

### 何をする？
論文の情報を **関係図として可視化**。

### どんな関係を見つける？
- 「TP53遺伝子」→ **抑制する** →「がん細胞の増殖」
- 「薬A」→ **阻害する** →「酵素B」
- 「治療法X」→ **改善する** →「症状Y」

### イメージ
![知識グラフのイメージ]
```
  [TP53] ----抑制---→ [がん]
     ↑                  ↓
  活性化             転移
     ↑                  ↓
  [薬A] ←---治療--- [抗がん剤]
```

マインドマップみたいに、情報のつながりが見える！

## 🧪 EPIC-7: Design（設計）- 実験提案担当

### 何をする？
「次にどんな研究をすべきか」を **提案する**。

### 4つの機能

| 機能 | 何をする？ |
|------|----------|
| Gap Analyzer | 「まだわかってないこと」を見つける |
| Experiment Designer | 「こんな実験どう？」と提案 |
| Protocol Generator | 実験の手順書を作成 |
| Statistical Designer | 「何人必要か」を計算 |

### 例
```
ギャップ発見: 「この薬の長期効果はまだ不明」
     ↓
実験提案: 「5年間の追跡調査」
     ↓
手順書: 「1. 100人集める 2. 毎年検査 3. 統計分析」
     ↓
統計設計: 「有意差を出すには最低80人必要」
```

## 🛡️ EPIC-9-10: Ops（運用）- 管理担当

### 何をする？
システムが **ちゃんと動いてるか監視**。

### 3つの機能

| 機能 | 何をする？ | イメージ |
|------|----------|---------|
| Audit Logger | 全ての動きを記録 | 防犯カメラ |
| Quality Gates | 品質チェック | テストの合格ライン |
| Cache Optimizer | 処理を高速化 | よく使うデータを覚えておく |

### 品質ゲート（守るべきルール）
- ✅ **根拠付け率 ≥ 95%** - ほぼ全ての主張に証拠が必要
- ✅ **根拠なし事実 = 0件** - 証拠なしで「〜である」と言うの禁止
- ✅ **パイプライン完走率 = 100%** - 途中でエラーになったらダメ

## 💡 なぜこれが必要？

研究者は毎日何百もの論文を読む必要があります。
でも人間には限界がある...

JARVISがあれば：
1. **時間節約**: 100本の論文を1時間で処理
2. **見落とし防止**: AIは疲れない
3. **客観性**: 感情に左右されない評価
4. **根拠重視**: 必ず証拠付きで説明
