# PARA (Parallel Agents)

## 使うとき
- 異なる故障領域/テスト群が独立

## 手順
1) 失敗を領域でクラスタ
2) 各領域にフォーカスドプロンプト
   - 目的/制約/禁止範囲/期待出力
3) 並列ディスパッチ
4) 結果レビュー→統合

## 注意
- 関連がある問題は分割しない
- 1領域=1責務、横断変更禁止
