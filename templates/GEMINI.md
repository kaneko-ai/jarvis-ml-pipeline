# Code Style Guidelines
- Follow existing module conventions and formatting.
- Keep functions small and focused.
- Prefer descriptive variable names.

# Documentation Requirements
- Update relevant README or docs when behavior changes.
- Document public APIs with docstrings.
- Record assumptions and limitations.

# Testing Standards
- Add or update tests for new features.
- Run the full test suite before merging.
- Document any skipped tests with rationale.
